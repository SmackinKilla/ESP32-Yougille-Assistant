# Шпаргалка по отрисовке на TFT (ST7735) и OLED (SSD1306)

Обе библиотеки наследуются от `Adafruit_GFX`, поэтому **90% функций одинаковые**. Различия — только в цветах и специфичных для OLED командах.

---

## 1. Базовая настройка экрана (вызывается в `setup()`)

| Функция | Описание |
|---------|----------|
| `tft.initR(INITR_BLACKTAB)` | **Только TFT.** Пробуждает контроллер. Параметры: `INITR_BLACKTAB`, `INITR_GREENTAB`, `INITR_REDTAB` — зависят от цвета вкладки на шлейфе |
| `tft.setRotation(0..3)` | Поворот экрана. 0 — портрет, 1 — альбом, 2 — перевёрнутый портрет, 3 — перевёрнутый альбом. **Меняет местами X и Y!** |
| `tft.fillScreen(цвет)` | Заливает весь экран одним цветом. **Обязательно вызывай после инициализации**, иначе будет мусор |
| `oled.begin(SSD1306_SWITCHCAPVCC, 0x3C)` | **Только OLED.** Инициализация. `0x3C` — I2C адрес (бывает `0x3D`) |
| `oled.clearDisplay()` | **Только OLED.** Очищает **буфер в памяти**, но НЕ отправляет на экран |
| `oled.display()` | **Только OLED. КРИТИЧЕСКИ ВАЖНО!** Отправляет буфер на экран. Без этой функции ты **ничего не увидишь** |

> ⚠️ **Главное отличие:** У TFT каждая команда отрисовки сразу летит на экран по SPI. У OLED сначала рисуешь в буфер (`clearDisplay`, `drawLine` и т.д.), а потом **одним вызовом** `display()` отправляешь всё на экран.

## 2. Цвета

### TFT (ST7735) — 16-битные цвета (RGB565)
Константы для использования:
- `ST7735_BLACK` — чёрный (0x0000)
- `ST7735_WHITE` — белый (0xFFFF)
- `ST7735_RED` — красный
- `ST7735_GREEN` — зелёный
- `ST7735_BLUE` — синий
- `ST7735_CYAN` — голубой
- `ST7735_MAGENTA` — пурпурный
- `ST7735_YELLOW` — жёлтый
- `ST7735_ORANGE` — оранжевый

Свой цвет можно создать через `tft.Color565(r, g, b)`, где r, g, b — от 0 до 255.

### OLED (SSD1306) — монохром (только 2 цвета!)
- `SSD1306_WHITE` — пиксель включён (горит)
- `SSD1306_BLACK` — пиксель выключен (не горит)
- `SSD1306_INVERSE` — инвертировать существующий пиксель

## 3. Примитивы (одинаковые для TFT и OLED)

| Функция | Параметры | Что делает |
|---------|-----------|------------|
| `drawPixel(x, y, color)` | координаты, цвет | Один пиксель |
| `drawLine(x0, y0, x1, y1, color)` | начало, конец, цвет | Линия между двумя точками |
| `drawRect(x, y, w, h, color)` | верхний левый угол, ширина, высота, цвет | Контур прямоугольника |
| `fillRect(x, y, w, h, color)` | то же | Залитый прямоугольник |
| `drawRoundRect(x, y, w, h, r, color)` | + радиус скругления | Контур скруглённого прямоугольника |
| `fillRoundRect(x, y, w, h, r, color)` | то же | Залитый скруглённый прямоугольник |
| `drawCircle(x, y, r, color)` | центр, радиус, цвет | Контур круга |
| `fillCircle(x, y, r, color)` | то же | Залитый круг |
| `drawTriangle(x0, y0, x1, y1, x2, y2, color)` | три вершины, цвет | Контур треугольника |
| `fillTriangle(x0, y0, x1, y1, x2, y2, color)` | то же | Залитый треугольник |

> ⚠️ **Координаты:** X и Y начинаются с **0**, а не с 1! Максимальный X = `SCREEN_WIDTH - 1`, максимальный Y = `SCREEN_HEIGHT - 1`.

## 4. Текст — это НЕ одна команда!

Текст выводится **последовательностью из 4 команд**. Каждая команда настраивает своё:

| Функция | Параметры | Что делает |
|---------|-----------|------------|
| `setCursor(x, y)` | координаты левого верхнего угла первой буквы | Устанавливает, **где** начнётся текст |
| `setTextColor(color)` | цвет (для TFT можно второй параметр — цвет фона) | Устанавливает **цвет** букв |
| `setTextColor(color, bgColor)` | цвет текста, цвет фона | **Только для TFT.** Позволяет затирать фон под текстом |
| `setTextSize(1..10)` | множитель размера | Устанавливает **размер** (1 — стандартный 6x8 пикселей, 2 — в 2 раза больше и т.д.) |
| `setTextWrap(true/false)` | флаг | Переносить ли текст на новую строку, если не влезает |
| `print(строка)` / `println(строка)` | текст, число, что угодно | **Выводит текст.** `println` добавляет перевод строки |

**Важно:** После `println` курсор автоматически переходит на новую строку. Если хочешь рисовать в том же месте — снова вызывай `setCursor`.

## 5. Специфичные функции OLED

| Функция | Параметры | Что делает |
|---------|-----------|------------|
| `clearDisplay()` | — | Очищает **буфер** (не экран!) |
| `display()` | — | Отправляет буфер на экран. **Без неё ничего не видно** |
| `invertDisplay(true/false)` | флаг | Инвертирует весь экран (белое становится чёрным и наоборот) |
| `startscrollright(0x00, 0x0F)` | начальная страница, конечная | Запускает аппаратную прокрутку вправо |
| `stopscroll()` | — | Останавливает прокрутку |

## 6. Полезные утилиты

| Функция | Что делает |
|---------|------------|
| `width()` | Возвращает текущую ширину экрана (с учётом `setRotation`) |
| `height()` | Возвращает текущую высоту экрана |
| `getRotation()` | Возвращает текущий поворот (0-3) |
| `drawBitmap(x, y, bitmap, w, h, color)` | Рисует монохромную картинку из PROGMEM (массива байт) |

## 7. Типичные ошибки

1. **Забыл `display()` на OLED** — рисуешь в пустоту
2. **Точка с запятой в `#define`** — ломает препроцессор
3. **Координаты `SCREEN_WIDTH` вместо `SCREEN_WIDTH - 1`** — пиксель обрезается
4. **Не вызвал `fillScreen()` после инициализации** — на экране мусор
5. **Забыл `setCursor()` перед `print()`** — текст рисуется в случайном месте (там, где курсор остался после прошлого вывода)
6. **Неправильный таб у TFT** — цвета инвертированы или сдвинуты. Попробуй `INITR_GREENTAB` или `INITR_REDTAB`
7. **Рисуешь на OLED без `clearDisplay()`** — новый текст накладывается на старый
